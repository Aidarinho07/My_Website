# CSS Structure and Naming

Проект использует модульную структуру CSS + BEM-подход.

## Порядок слоев (важен для каскада)

1. `00-base.css` — токены, reset, базовая типографика и контейнер.
2. `01-header.css` ... `07-footer.css` — базовые блоки по зонам интерфейса.
3. `08-responsive-legacy.css` — базовая адаптивность.
4. `09-premium-overrides.css` — визуальные апгрейды и расширенные блоки.
5. `10-reveal.css` — анимации появления.
6. `11-mobile-ux.css` — мобильное UX-поведение (меню, нижняя панель, keyframes).
7. `12-lightbox.css` — стили модального просмотра документов.
8. `13-soft-feminine.css` — финальный тематический слой (polish).

`style.css` — только entrypoint c `@import` в фиксированном порядке.

## Методология именования классов (BEM)

- `block` — самостоятельный компонент: `.hero`, `.card`, `.footer`.
- `block__element` — часть блока: `.hero__title`, `.footer__link`.
- `block--modifier` — состояние/вариант: `.button--primary`, `.plan--featured`.
- `is-*` — JS-состояния: `.is-open`, `.is-active`, `.is-visible`.
- `u-*` (опционально) — утилиты, если появятся: `u-hidden`, `u-text-center`.

## Канонические имена (после миграции)

- `hero__quote`
- `list-bullets`
- `section-note`
- `about__grid`
- `layout-grid--2`
- `text-muted`

Legacy-алиасы удалены. В новых изменениях используем только канонические имена.

## Практические правила

- Новый компонент добавляй в файл своей зоны (`header/footer/cards/...`).
- Если правило влияет только на мобильный UX, клади в `11-mobile-ux.css`.
- Тематические визуальные правки (цвет/градиент/мягкие тени) — в `13-soft-feminine.css`.
- Не дублируй селектор в нескольких файлах без причины: приоритет должен быть осознанным.
